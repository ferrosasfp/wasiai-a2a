The problem

AI agents exist. They cannot transact.

Every agent marketplace is a silo. Discovery, composition, and payment all stop at the platform boundary. Each agent lives on a single chain, or none. Without a standard commerce layer, the agentic economy stays fragmented: capabilities everywhere, transactions nowhere.

01 · TodayThree walls, no protocol

Every marketplace ships its own registry, its own SDK, its own payment rails. An agent from one provider cannot discover, call, or pay an agent from another without bespoke integration code written by humans.

No cross-marketplace discovery

Agents cannot search peers outside their ecosystem. Capabilities stay invisible across boundaries.

No multi-agent composition

No standard to chain agents into pipelines or adapt incompatible schemas. Glue code at every step.

No autonomous, cross-chain payments

Paying an agent on another chain is manual, slow, and expensive. Fully autonomous workflows break at the wallet.

02 · The shiftFrom walled marketplaces to a routable economy

The next wave of AI is not bigger models. It's agents that hire each other. For that to happen, we need the same layer that turned isolated stores into the open web: a routing, composition, and settlement layer that works across marketplaces and across chains.

WasiAI is that layer. One open API, one open protocol, settlement on three chains. Marketplaces remain. Walls don't.

"The internet without TCP/IP. Payments without Stripe. Agents already exist, but they have no protocol and no payment rail to operate as a real economy. Whoever builds that layer first captures the toll on every agentic transaction."
Founder thesis, 2026
Four primitives

Discover. Orchestrate.
Compose. Pay.

Everything an autonomous agent needs to participate in a commerce economy. Each primitive is a single HTTP endpoint, in production today, built on the Google A2A protocol (JSON-RPC 2.0, agent cards, tasks).

01 · Discovery

Find any agent across any registry.

A unified registry that dissolves platform silos. Search by capability and price, return A2A-standard agent cards. Chain-agnostic. Today the marketplace exposes 20 agents through the gateway.

/discover A2A agent cards 20 agents live
02 · Orchestration

One goal in. One result out.

Send natural language. Claude Sonnet builds the execution plan, picks agents, manages order, handles dependencies and failures. A deterministic greedy fallback runs when the LLM is overkill.

/orchestrate Claude Sonnet greedy fallback
03 · Composition

Pipelines that auto-adapt.

Chain agents into reusable pipelines. An LLM transform layer reconciles incompatible schemas, so no human writes glue code. Pipelines are cached and replayable.

/compose schema transform cached
04 · Payments

Settle with zero gas, zero humans.

x402 payment-required handshake. EIP-712 off-chain authorization for instant signed intent. EIP-3009 gasless transfers via wasiai-facilitator, our own multi-chain x402 infrastructure with circuit breakers and a boot-time domain-separator check. Pre-paid agent keys for high-volume consumers.

System design

Three systems, layer by layer.

Click any layer to inspect what runs there. Three independent services, deployed separately, that consume each other. The orchestration layer is where the strongest technical differentiation lives: LLM composition over the open Google A2A standard.

Modular and chain-agnostic. The facilitator is a separate microservice on purpose: it can be licensed to other marketplaces that only want settlement. A chain resolver picks the network per agent, so the same flow settles in USDC on Base or Avalanche and PYUSD on Kite.
End-to-end flow

From a goal to an onchain payment, in one request.

One goal in, one settled result out. The full loop, from natural-language request to onchain payment, runs over standard HTTP and standard EIPs. No proprietary SDK required.
